# 2026-06-23 PATCS 概率云核心计划

## 背景

当前旧版 PATCS artifact 使用 stage-first 的 `[S, N, P, D]` cloud。根据最新算法讨论，这不是最终目标。新的核心应是：

```text
demo_i-conditioned probabilistic trajectory tube
```

即每个目标 demo_i 都有自己的监督云。其他同任务 demo 的完整轨迹先缩放/对齐到 demo_i 的长度，再围绕 demo_i 的夹爪转换 observation 做事件收束。

## 当前原型定义

对目标 demo_i：

```text
aligned_points: [N, T_i, 3]
target_xyz:     [T_i, 3]
transitions:    [K_i]
```

每个时间点 `t` 不再只保存点云，而是建立一个局部概率截面：

```text
local_frame[t] = [tangent, normal_1, normal_2]
mean[t]        = target_xyz[t] 或 aligned cloud mean 与 target blend
cov2[t]        = local normal plane 上的 2D covariance
scale[t]       = event black-hole contraction scale
```

训练时的 tube loss 可以先用 Gaussian NLL / Mahalanobis：

```text
loss_t = -log N(project(pred_ee_t - mean[t]), cov2[t])
```

后续可替换成 KDE/GMM log-density，但第一阶段先用 Gaussian 截面，理由：

- 依赖少，只需 numpy/scipy。
- 数值稳定，容易接入 torch loss。
- 可直接生成桶状三维曲面用于可视化。
- 与 action chunk 训练中的 per-step supervision 兼容。

注意：这一版是快速原型，不是最终核心。它的计算路径是：

```text
每个 phase 先算 2D normal-plane Gaussian 截面
再把截面沿轨迹拼接成 3D tube surface
```

因此天然可能出现截面拼接感、局部 frame 扭转、边缘不顺滑等问题。它只用于 synthetic 调试和早期 smoke，不应作为最终 PATCS 概率云核心。

## 下一阶段核心定义：先算完整 3D 概率团，再做收束场

最终方向应改成两层：

### 1. 大概率团

先从所有同任务 demo 的完整轨迹点一次性估计连续 3D 概率团：

```text
p(x, y, z, phase)
或
p(x, y, z | phase)
```

候选实现：

- `global_3d_gaussian`
- 3D KDE
- GMM
- voxel density + marching cubes / isosurface

这一层输出应是连续概率体或可查询 log-density，而不是一组独立 2D 截面。

### 2. demo_i 条件收束场

在大概率团基础上，再按当前 demo_i、夹爪 transition 和 event 做条件变形：

```text
global_3d_density
-> demo_i-conditioned contraction
-> olive envelope
-> precision narrow channel
-> final black-hole anchor
```

也就是说，收束不应该通过重算每个截面概率来完成，而应该作为一个连续场作用在大概率团上。

## 精细操作通道

当前 `blackhole_window` 太短时，只会表现成点状强约束：

```text
transition ±2 timestep
```

目标应拆成：

```text
wide cloud -> olive narrowing -> narrow precision channel -> final black-hole anchor
```

建议配置：

```text
density_mode = "global_3d_gaussian" | "phase_2d"
precision_channel_window = 8~20
precision_channel_radius
blackhole_window = 1~3
event_radius
```

核心半径/密度逻辑应改成：

```text
radius(t) =
  base_global_density_radius
  * olive_scale(t)
  * channel_scale(t)
  + event_radius
```

其中 `channel_scale(t)` 在 transition 两侧较长范围内逐渐变小，形成狭窄但连续可走的精细操作通道；真正靠近 transition endpoint 时才进入 final black-hole anchor。

## 黑洞收束

夹爪转换点附近必须向当前 demo_i 的 observation 收束。定义：

```text
u = min_distance_to_transition / event_window
u in [0, 1]
scale = u ** contraction_power
```

当 `contraction_power > 1` 时，远离转换点开始收缩快，越接近转换点收缩越慢；在转换点处半径压到 `event_radius`。

最终 covariance：

```text
cov2_contracted[t] = scale[t]^2 * cov2_base[t] + event_radius^2 * I
```

转换点处：

```text
mean[t] = target_xyz[t]
cov2[t] ~= event_radius^2 * I
```

## 可视化目标

单个 demo 输出三图：

1. 三维概率桶状云：由连续 Gaussian 截面连接成半透明 surface，显示密度 tube。
2. phase-separated 截面流向图：每个 phase 截面分开画，包含截面椭圆/密度和局部 EE 方向箭头。
3. target demo phase 轨迹：目标 demo 彩色显示 phase，其他 demos 以 0.2 透明度参考。

测试模式允许一次输出三个同任务 demo 的三行对比图。

## 实施顺序

1. 保留当前 `phase_2d` Gaussian tube 作为 synthetic 原型和回退实现。
2. 新增 `density_mode="global_3d_gaussian"`，先在 synthetic 直线数据上调出连续概率团。
3. 增加 `precision_channel_window / precision_channel_radius / blackhole_window / event_radius`。
4. 实现 `global_3d_density -> olive contraction -> precision channel -> final black-hole` 可视化。
5. 再将该逻辑推广到真实 demo 数据。
6. 最后接入训练 loss，替代/并行旧 hull PATCS loss。

## 当前阶段边界

本阶段先实现核心计算与可视化，不直接替换训练主链路。旧 artifact 与训练接口暂时保留，避免大规模重构。
